Report¶

1. Prediction calculation, 2. LIME decomposition and 3. Decompositions comparison¶

A model (Random Forest) was first evaluated on some random observations. The predicted probabilities of the patient being healthy were: 0.99, 0.19, 0.86, 0.92, The explanations are presented below. The model seems quite stable - a given feature contributes the same way to the output in different samples, if that feature's value is similar.

4. Comparison: LIME and SHAP¶

Below, SHAP explanations for the same samples as before are presented. The most obvious difference is that SHAP attributions need to add up to model's output. SHAP and LIME seem consistent on the given examples (a given feature contributes positively in both models or it contributes negatively in both). The feature importance however is different.

5. SHAP on a different model¶

SHAP was also used to explain a Logistic Regression model. Features are ranked differently than with Random Forest.

Dataset contains two categorical variable: cp and restecg. Those are one-hot encoded in preprocessing. All models used (Logistic Regression, Decision Tree, Random Forest) can benefit from one-hot encoding the categorical variables.

In [ ]:
def to_one_hot(data, column):
    dummies = pd.get_dummies(data[column])
    for dummy_value in dummies.columns:
        data = pd.concat([data, dummies[dummy_value].rename(f'{column}_{dummy_value}')], axis=1)
    return data.drop(column, axis=1)

data = to_one_hot(data, 'cp')
data = to_one_hot(data, 'restecg')

X = data.drop(columns=['output'])
y = data['output']
